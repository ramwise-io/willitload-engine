# `willitload` Sample Outputs & Scenario Reports

This document provides a gallery of sample terminal outputs and JSON payloads generated by `willitload` across different structural scenarios. These can be used to understand the CLI display format and JSON contract.

## 🛡️ Scenario 2: `willitload check --align name --extra strict`

Checking a fileset against a baseline schema under **Name Alignment** where extra columns are rejected.

### Command
```bash
willitload check ./data/raw_imports/ --against ./baseline.schema --align name --extra strict
```

  [X] ./data/raw_imports/orders_extra_region.csv
      ERROR column 'region': Column 'region' is present in this file but not declared in the baseline. With --extra strict, this is treated as drift. (EXTRA_COLUMN)

  [X] ./data/raw_imports/orders_missing_notes.csv

## 🚫 Scenario 3: Headerless File in Name Mode (Impossible Quadrant)

Running name-bound validation against a fileset containing a headerless file. `willitload` guards this boundary and fails the check rather than quietly switching to position matching.

### Command
```bash
willitload check ./data/raw_imports/ --against ./baseline.schema --align name
```

### Terminal Output
```ansi
willitload check -- ./data/raw_imports/
  Baseline: Flat schema file: ./baseline.schema
  Mode: name  |  Extra-column policy: strict
  1 files seen  |  0 golden  |  0 warned  |  1 broken
  Elapsed: 15ms

BROKEN - 1 file(s) do not conform

  [X] ./data/raw_imports/raw_headerless_data.csv
      ERROR file header: Name-mode alignment requires named columns, but this file has no header. Cannot bind by name -- reported as anomaly, not silently switched to position mode. (HEADERLESS_NAME_MODE)
```

## 🔄 Scenario 4: `willitload check --align position` (Polybase Style)

Checking in **Position Mode** (ordinal matching) where name drift is tolerated as warnings (swap-hints) but column count changes are breaking.

### Command
```bash
willitload check ./data/raw_imports/ --against ./baseline.schema --align position
```

### Terminal Output
```ansi
willitload check -- ./data/raw_imports/
  Baseline: Flat schema file: ./baseline.schema
  Mode: position  |  Extra-column policy: strict
  1,004 files seen  |  1,000 golden  |  3 warned  |  1 broken
  Elapsed: 1120ms

BROKEN - 1 file(s) do not conform

  [X] ./data/raw_imports/orders_missing_notes.csv
      ERROR column count: Expected 5 columns, but found 4. Position-bound loads map strictly by position and require exact count matching. (COUNT_CHANGED)

3 file(s) with warnings

  [!] ./data/raw_imports/orders_renamed_total.csv
      WARN column at position 2: Column name 'total_amount' at position 2 differs from baseline expectation 'amount'. Position-bound loads tolerate name drift, but this may hint at a column swap. (COLUMN_NAME_MISMATCH)
```
